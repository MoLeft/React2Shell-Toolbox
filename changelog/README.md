# Changelog 目录说明

## 📝 目录用途

此目录用于存放每个版本的更新说明文件。当用户检查更新时，应用会优先从这里读取对应版本的更新说明。

## 📋 文件命名规范

- 文件名格式：`v{版本号}.md`
- 示例：`v1.0.0.md`、`v1.0.1.md`、`v2.0.0.md`

## ✍️ 编写规范

每个版本的更新说明应包含以下内容：

```markdown
# v{版本号} 更新说明

## 🎉 新功能

- 功能1描述
- 功能2描述

## 🐛 Bug 修复

- 修复1描述
- 修复2描述

## 🔧 技术改进

- 改进1描述
- 改进2描述

## ⚠️ 破坏性变更（如有）

- 变更1描述
- 变更2描述

## 📝 使用说明

其他需要用户注意的事项
```

## 🔄 更新流程

1. 在此目录创建新版本的 markdown 文件
2. 编写详细的更新说明
3. 更新 `package.json` 中的版本号
4. 提交代码并创建 GitHub Release
5. 用户检查更新时会自动显示对应版本的更新说明

## 💡 注意事项

- 更新说明支持 Markdown 格式
- 建议使用 emoji 图标增强可读性
- 如果本地没有对应版本的文件，会自动使用 GitHub Release 的说明
- changelog 目录会被打包到应用中，确保用户能离线查看更新说明
